# Native Trilingual Content Bundles Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Generate every scheduled concept as independently authored, native-quality `NL-BE`, `FR-BE`, and `EN` drafts with separate approval and publication.

**Architecture:** Persist one factual content-bundle brief, fan it out to three locale-native writers, validate every output with existing brand/schema gates plus a deterministic locale gate, and enqueue passing variants independently. Scheduler and catch-up commands are idempotent per date/action/bundle/locale; Meta publication additionally validates locale-specific targeting.

**Tech Stack:** Python 3.12, Anthropic client, SQLAlchemy/PostgreSQL, pytest, Slack outbox, Meta Business SDK, GitHub Actions.

## Global Constraints

- NL/FR variants are authored natively, never translated from English source copy.
- Each locale is independently generated, scored, revised, approved, and published.
- Locale gate categories grammar, idiomaticity, locale fit, and literal-translation risk must each score at least 85 with no critical grammar error.
- A failing locale is retried at most twice; passing/decided locales are never regenerated by schedule retry.
- Paid budget is never shared, increased, or approved across locales implicitly.
- Draft 156 remains archived and is never reused or automatically replaced.
- Deployment and today's catch-up run only after GitHub CI succeeds.
